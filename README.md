# LightOS Orion — RISC-V Kernel

A clean, royalty-free, UNIX-style operating system kernel written from
scratch in Rust (`no_std`, zero external crates), targeting **RISC-V
RV64GC**. LightOS is the host OS layer for LightRail AI's Neural
Compute Element (NCE) hardware: minimal, deterministic, and
AI-workload-aware from the ground up.

*Neural-native. Royalty-free. RISC-V first.*

## Install & run

LightOS runs under QEMU's RISC-V emulator — no hardware needed.

**Linux / macOS / WSL** (bash — *not* Windows PowerShell), once a
[release](https://github.com/Lightiam/LightOS_Orion_RISC-V/releases) is
published:

```sh
# install QEMU first: apt-get install qemu-system-misc  /  brew install qemu
curl -fsSL https://raw.githubusercontent.com/Lightiam/LightOS_Orion_RISC-V/main/scripts/install.sh | bash
lightos
```

**Works today from source** on any of the above (needs Rust, QEMU,
make, python3, fsck.minix):

```sh
git clone https://github.com/Lightiam/LightOS_Orion_RISC-V
cd LightOS_Orion_RISC-V && make run
```

You boot straight to an interactive shell. **Windows** users: use WSL,
or native QEMU for Windows with the bundle's `run.ps1`. Full details
including the Windows path in **[INSTALL.md](INSTALL.md)**. Exit the
emulator with `Ctrl-A` then `x`.

## What works today

Boot LightOS in QEMU and you get: a supervisor-mode kernel with Sv39
paging, preemptive multitasking of real user-mode processes, a
POSIX-flavoured syscall surface, a Minix3 root filesystem on
virtio-blk, an interactive shell, and an NCE hardware abstraction
layer exposed as `/dev/nce*` devices.

| Phase | Deliverable | Status |
|-------|-------------|--------|
| 0 | Bare boot: M→S transition, NS16550A UART, QEMU virt | done |
| 1 | Page-frame allocator, kernel heap (`Box`/`Vec`), Sv39 MMU | done |
| 2 | Trap handling, 10 ms Sstc timer tick, PLIC, IRQ UART | done |
| 3 | Processes: fork/exec/exit/wait, preemptive round-robin | done |
| 4 | Syscalls (Linux riscv64 ABI numbering) + libc shim | done |
| 5 | VirtIO block driver, Minix3 root FS, VFS, fd table | done |
| 6 | Interactive shell: cd, pwd, ls, cat, echo, externals | done |
| 7 | NCE HAL: descriptors, power states, affinity, /dev/nce* | done |
| 8 | Networking: virtio-net + Ethernet/ARP/IPv4/ICMP, boot ping | done |
| 9 | UDP + userspace socket API (socket/bind/sendto/recvfrom) | done |
| 10 | TCP client (connect/send/recv) — HTTP GET from userspace | done |

Every phase is verified by an end-to-end QEMU boot test
(`scripts/boot_test.sh`) that boots the kernel on emulated hardware,
drives a scripted interactive console session, and asserts all 36
milestone markers — including preemption interleaving and NCE power
state transitions.

## Building and running

Requirements: Rust (stable, via rustup — `rust-toolchain.toml` pins
everything), `qemu-system-riscv64`, GNU make, python3, and
`fsck.minix` (util-linux) for root-image validation.

```sh
make build   # userspace ELFs, kernel, (see also: make disk)
make run     # boot to the interactive shell in QEMU
make test    # full end-to-end boot test (35 assertions)
make release # produce dist/lightos-<version>.tar.gz bundle
make gdb     # boot halted with a GDB stub on :1234
```

Or with nothing but Docker:

```sh
make docker-test   # = docker build -t lightos . && docker run --rm lightos
```

Exit QEMU with `Ctrl-A x`.

```

  main.rs        mstart (M-mode init) + kinit (S-mode entry)
  uart.rs        NS16550A driver, IRQ-driven receive
  trap/          trap vector, PLIC, CLINT/Sstc timer, TrapFrame
  mem/           page frames, heap, Sv39 MMU, user-copy guards
  sched/         PCBs, user context switch (switch.S), round-robin
  syscall/       ecall dispatch + POSIX implementations
  fs/            VFS, Minix3 driver, embedded ramfs fallback
  drivers/       VirtIO MMIO transport + block device
  nce/           NCE HAL: descriptors, power states, affinity
userspace/       cargo workspace: libc_shim, init, sh, hello,
                 ncectl, selftest
rootfs/          files installed into the Minix3 root image
scripts/         mkfs_minix3.py, boot_test.sh, make_splash.sh
```

## Design notes (v1)

- **Single scheduling hart, non-preemptible kernel.** The scheduler
  switches *trap frames*: a timer trap from user mode saves the full
  context, and `schedule()` srets into the next Ready process.
  Blocking syscalls park the caller; whoever wakes it writes the
  result into its trap frame. Secondary harts are parked until the
  SMP upgrade.
- **Address spaces** share the kernel's page-table subtrees (mapped
  without `PTE_U`), so traps need no trampoline page; user code lives
  in the 1 GiB slot at `0x4000_0000`.
- **Filesystem** is read-only Minix3 in v1; the image is generated by
  `scripts/mkfs_minix3.py` and cross-validated with `fsck.minix`.
- **NCE slots** are enumerated from the device tree
  (`compatible = "lightrail,nce"`); QEMU has none, so emulated slots
  keep the `/dev/nce*` surface and power state machine exercisable
  before silicon.

## Post-v1 roadmap (NCE / AI workload extensions)

| Feature | Description |
|---|---|
| `lightos-ray` | Kernel-bypass path for Ray task scheduling; `/dev/ray_shm` shared memory channel |
| `nce_affinity` | SMP scheduler consumes affinity hints to pin Ray workers to NCE-adjacent harts |
| `hugepage_nce` | 2 MiB huge pages for NCE DMA buffers; minimize TLB pressure |
| `power_governor` | Per-NCE DVFS policy daemon |
| `dtb_nce_enum` | Full FDT property parse of NCE nodes; auto-configure MMIO ranges |
| `fault_tolerance` | Kernel watchdog: detect NCE hangs, reset and re-init without full reboot |
